下面给你一份**“严格基于 Anduin2017/HowToCook 仓库内容对齐的 eval_set_v2 设计稿”**。
我做了三件事来保证**不会出现“库中不存在却要求 AUTO”**的问题：

* **AUTO / AMBIG / 类目冲突**：只使用 **HowToCook 仓库中明确存在的菜名或目录能力**
* **LOW / OOD / 空泛**：明确是 **库中不存在 / 不可回答 / 不应推荐**
* 所有 query 都符合你当前 **定版策略（title_hit / 推荐型 / LOW gate）**

你可以直接转成 `jsonl`，我先用**人类可读清单**给你，方便你快速过一遍。
